# 页面拓扑清单

> 与 `frontend/src/router/RouterApp.tsx`（React Router v6 + Next.js App Router 外壳）一致。

## 业务模块划分

- 认证模块：`/login`（未包裹 `AppLayout`，与需登录的 `/app/*` 分离）
- 工作台模块：`/app/dashboard`
- 历史模块：`/app/history`（外层带 `RoutePermissionGuard requiredRole="user"`）
- Wiki 模块：`/app/wiki/:taskId`
- 兜底模块：`*` → 404

## 页面跳转关系

- `/` → `RestoreLastPath`：读取 `localStorage` 中 `wiki_route_tracking` 的 `path`；无有效记录或未登录且路径为 `/login` 等 → `/app/dashboard`
- `/login` → 登录成功后跳转 `next` 查询参数或默认 `/app/dashboard`（以 `LoginPage` 实现为准）
- `/app/dashboard` → 任务完成后可编程式跳转 `/app/wiki/:taskId?repo=...`（以 `DashboardPage` 为准）
- `/app/history` → 点击任务/会话跳转 `/app/wiki/:taskId` 或带 `chatId` 等查询参数（以 `HistoryPage` 为准）
- 任意未匹配路径 → `*` → `NotFoundPage`

## 守卫拓扑

- `GlobalRouteGuard`：路由埋点与最近访问路径写入 `wiki_route_tracking`
- `AuthGuard`：未登录访问受保护路由时重定向到 `/login`
- `RoutePermissionGuard`：按 `requiredRole`（如 `user`）限制子路由（当前用于 `history` 分支）
- 组件内：`WikiRoute` / `WikiViewer` 等在数据未就绪时的加载与错误处理（若存在 `ComponentDataGuard` 类逻辑，以对应视图代码为准）

## 布局与路由视图映射

- `RootLayoutView`：根级容器与主题背景，挂载 `Outlet`
- `AppLayout`：`Header` + `Sidebar` + 主 `Outlet`（`/app/dashboard`、`/app/history`、`/app/wiki/:taskId` 均在其下）
- Next.js：`src/app/page.tsx` 与 `src/app/[...slug]/page.tsx` 仅负责客户端挂载 `RouterApp`，便于单页内深链接

## 与后端交互（摘要）

- API 基址：`NEXT_PUBLIC_API_URL`（`frontend/src/lib/api.ts`），默认 `http://localhost:8000`
- 主要能力：任务生成与轮询、RAG/Agent 聊天（含 SSE）、会话历史、文件内容拉取、用户 `profile`（语言/主题）
